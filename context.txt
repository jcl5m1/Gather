# Gather - Space Simulation Game

This is a space simulation game written in TypeScript. It is an npm project with auto-reloading development server.

## Project Structure

### Main Application (src/moonOrbitSim/)
The primary codebase for the orbital mechanics simulation.

**Core Files:**
- `index.ts` - Application entry point, initializes the app
- `app.ts` - Main application setup and initialization
- `gameLoop.ts` - **MAIN GAME LOOP** - handles rendering, physics updates, and game state
- `config.ts` - Loads and parses configuration from config.json with unit conversion
- `config.json` - **ALL STATIC VALUES** - physics constants, scene settings, body definitions
- `index.html` - Entry HTML file

**Game Objects:**
- `orbitalBody.ts` - OrbitalBody class (extends Body) with physics update logic (numerical/analytical/bezier), dual-rendering support, and dynamic LOD rendering (mesh vs dot)
- `trajectory.ts` - Trajectory class for orbit calculation (elliptical/hyperbolic) and visualization. **Includes Rational Bezier logic** (BezierCurve, TrajectoryRenderer).
- `transferTrajectory.ts` - Manages calculation and visualization of transfer orbits
- `types.ts` - Body base class; JSON serialization utilities; OrbitControls interface

**Simulation & Physics:**
- `simulationController.ts` - Controls time scaling and simulation state
- `orbitUtils.ts` - Utility functions for orbital calculations (Kepler, Bezier generation) and **TransferCalculator** for maneuvers
- `gpuOrbitCompute.ts` - GPU-accelerated massive batch orbit position computation using Data Textures and Shaders (GPGPU)

**UI & Interaction:**
- `uiManager.ts` - UI rendering and **formatters for unit display**
- `commandProcessor.ts` - **TEXT COMMAND INTERFACE** - allows automated execution of all UI actions
- `cameraManager.ts` - Camera controls, tracking, and free camera mode (WASD+RF movement)
- `uiWindow.ts` - Base UIWindow class for creating draggable, resizable windows
- `plotWindow.ts` - PlotWindow class (extends UIWindow) for 2D XY plotting
- `tooltipManager.ts` - Manages interactive tooltips for game objects
- `plotDemo.ts` - Creates analysis plots for bezier orbit approximation
- `propertyInspector.ts` - Property inspection and modification interface

**Utilities:**
- `units.ts` - Unit system (kilometers, kilograms, seconds, etc.)
- `unitsVector3.ts` - Vector3 with unit support

**Testing & Benchmarking:**
- `performanceTest.ts` - Benchmarking suite
- `velocitySymmetryTest.ts` - Verifies velocity calculations
- `testCommands.ts` - Command interface testing script
- `testLambert.ts` - **Lambert Solver Test** - Verification for orbital transfer math

**Documentation:**
- `COMMAND_INTERFACE.md` - Documentation for text commands

### Resource Gathering Game (src/mineGather/)
Secondary game module focusing on resource management.

- `gameManager.ts` - Core logic for mineGather
- `storageManager.ts` - Inventory and storage management
- `uiManager.ts` - UI specific to mineGather
- `types.ts` - Type definitions for resources and game state
- `index.ts` - Entry point for mineGather
- `index.html` - Entry HTML for mineGather

### Other Modules
- `src/index.ts` - Legacy code
- `src/renderbody.ts` - Legacy code
- `src/utils.ts` - General utilities
- `src/custom.d.ts` - Type declarations

## Key Concepts

### Orbit Geometry
- **Bezier Orbits**: Rational Bezier curves are used for high-performance orbit rendering and exact position calculation. This logic is centralized in `trajectory.ts` and `orbitUtils.ts`.
- All orbital math should be in the `orbitUtils.ts` file. The classes should not contain any orbital math.
- The only code using the analytical orbit math should be in orbit utils and used only to generate the bezier approximations. Everything else should use the bezier approximations for speed.

### Units
- **Internal**: kilometers, radians, kilograms, seconds
- **Display**: Use formatters in `uiManager.ts` to convert to user-friendly units

### Command-Driven Architecture
- **Text Commands**: Primary method for automated testing and precise control (see `COMMAND_INTERFACE.md`).

## Development

- **Testing**: Use text commands and browser console.
