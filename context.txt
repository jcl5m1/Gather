# Gather - Space Simulation Game

This is a space simulation game written in TypeScript. It is an npm project with auto-reloading development server.

## Project Structure

### Main Application (src/moonOrbitSim/)
The primary codebase for the orbital mechanics simulation.

**Core Files:**
- `index.ts` - Application entry point, initializes the app
- `app.ts` - Main application setup and initialization
- `gameLoop.ts` - **MAIN GAME LOOP** - handles rendering, physics updates, and game state
- `config.ts` - Loads and parses configuration from config.json with unit conversion
- `config.json` - **ALL STATIC VALUES** - physics constants, scene settings, body definitions

**Game Objects:**
- `orbitalBody.ts` - OrbitalBody class (extends Body) for orbital mechanics; OrbitalBodyRenderer class handles 3D rendering with automatic mesh/sprite switching based on screen size
- `trajectory.ts` - Trajectory class for orbit calculation and visualization; BezierCurve class for curve math; TrajectoryRenderer and BezierCurveRenderer classes for rendering orbit paths
- `types.ts` - Body base class (properties: position, velocity, mass, radius, name, color, etc.); JSON serialization utilities (Vector3fromJSON, Vector3toJSON, toJSON, fromJSON); OrbitControls interface

**Simulation & Physics:**
- `simulationController.ts` - Controls time scaling and simulation state
- `orbitUtils.ts` - Utility functions for orbital calculations

**UI & Interaction:**
- `uiManager.ts` - UI rendering and **formatters for unit display** (use these for displaying values!)
- `commandProcessor.ts` - **TEXT COMMAND INTERFACE** - allows automated execution of all UI actions
- `cameraManager.ts` - Camera controls, tracking, and free camera mode (WASD+RF movement)
- `uiWindow.ts` - Base UIWindow class for creating draggable, resizable windows with title bars
- `plotWindow.ts` - PlotWindow class (extends UIWindow) for 2D XY plotting with interactive hover tooltips, grid, and axes
- `plotDemo.ts` - Creates analysis plots for bezier orbit approximation; displays separate time warp and distance error plots
- `propertyInspector.ts` - Property inspection and modification interface for game objects

**Utilities:**
- `units.ts` - Unit system (kilometers, kilograms, seconds, etc.)
- `unitsVector3.ts` - Vector3 with unit support

**Documentation:**
- `COMMAND_INTERFACE.md` - Documentation for text commands

### Other Modules
- `src/mineGather/` - Separate game module (resource gathering game)
- `src/index.ts` - **DEPRECATED** legacy code, not used by current build
- `src/renderbody.ts` - **DEPRECATED** legacy code
- `src/utils.ts` - Legacy utilities

## Key Concepts

### Type Organization
- **Module-specific types**: Each module has its own `types.ts` file
  - `src/moonOrbitSim/types.ts` - Body base class, JSON serialization utilities, OrbitControls interface
  - `src/mineGather/types.ts` - Resource types, game state types
- **No shared types folder** - types are co-located with the modules that use them
- **Orbit geometry types** - defined in `trajectory.ts` alongside implementation (BezierCurve, TrajectoryParameters, TrajectoryType, etc.)

### Units
- **Internal storage**: kilometers, radians, kilograms, seconds, liters, kelvin, newtons, joules, watts
- **Display**: Use formatters in `uiManager.ts` to convert to user-friendly units

### Command-Driven Architecture
- **Never click UI widgets directly** - use text commands via `commandProcessor.ts`
- **All UI actions must go through commands** - ensures full automation capability
- **Test via console** - use browser console output to verify behavior, minimize screenshot processing

### Inspector
- UI has an inspector panel that organizes game data and allows inspection/modification of properties

## Development Workflow

1. **Don't run `npm build`** unless absolutely necessary - the dev server auto-updates
2. **Test in browser** after changes - check for compilation errors in browser console
3. **Use text commands** to test functionality - see `COMMAND_INTERFACE.md` for command reference
4. **Use console.log** for debugging - check browser developer console for output
5. **Test command behavior** if commands were modified before marking task complete
